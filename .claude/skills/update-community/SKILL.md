# Update Community

Communities (subdivisions) live in `src/data/communities.ts`. One file. Each community gets:
- A card on `/available-lots/`
- An auto-generated detail page at `/available-lots/<slug>/`
- A filtered lot inventory table on the detail page
- Plat map downloads (PDF + JPG preview)

The detail page is generated by `src/pages/available-lots/[slug].astro` — no need to create new page files when adding a community.

## Data file: `src/data/communities.ts`

### Community object schema

```typescript
{
  slug: 'hawks-landing',                 // URL slug, lowercase, hyphenated
  name: 'Hawks Landing',                 // Display name
  status: 'selling',                     // 'selling' | 'coming-soon'
  description: 'Bigger lots on...',      // 1-2 sentence card description (used on index)
  longDescription: 'Hawks Landing...',   // Multi-paragraph copy for the detail page
  image: '/media/.../featured.jpg',      // Card thumbnail on the /available-lots/ index
  heroImage: '/media/.../hero.jpg',      // Detail-page hero (optional — falls back to `image`)
  address: 'Foothills of Ammon, ID',     // General location
  highlights: [                          // 4-6 selling-point bullets
    { title: 'Sweeping valley views', body: '...' },
    ...
  ],
  gallery: [                             // Photo strip for the detail page
    { src: '/media/.../photo1.jpg', alt: '...' },
    ...
  ],
  platMaps: [                            // PDF plat maps
    {
      label: 'Division 4',
      pdf: '/documents/plat-maps/hawks-landing-div4.pdf',
      preview: '/media/plat-maps/hawks-landing-div4-preview.jpg',
      fileSizeMB: 3.6,
    },
    ...
  ],
}
```

### Status field

- `'selling'` — community is active, shows up on the index, detail page is fully rendered
- `'coming-soon'` — community is teased on the index but the detail page shows a "join the interest list" placeholder

### What's optional vs. required

**Required for any community:** `slug`, `name`, `status`, `description`, `image`.

**Required for `'selling'` communities with lots:** Also `longDescription`, `highlights`, and ideally `gallery` + `platMaps`.

**Optional for `'coming-soon'` communities:** Most fields except slug/name/status/description/image.

## Filtering behavior

The `/available-lots/` index page (in `src/pages/available-lots/index.astro`) automatically filters out communities that have no available lots. Specifically:

```typescript
const communitiesWithLots = communities.filter(c => lotsByCommunity(c.slug).length > 0);
```

So if a community has zero rows in `lots.ts` matching its slug, it won't appear on the index — even if it's set to `'selling'`. This is by design: a community fully sold out shouldn't waste real estate on the page.

**The Parks** and **Moser Estates** are examples — they're built-out communities that live ONLY on `/quick-move-ins/` (because Odyssey has finished homes there) but not on `/available-lots/`.

## Image assets

| Asset | Location | Aspect | Notes |
|---|---|---|---|
| Card thumbnail | `public/media/<existing-or-new>/` | 4:3 | Used on `/available-lots/` index card |
| Hero image | `public/media/communities/<slug>/hero.jpg` | 16:9 | Optional — falls back to thumbnail. Used on detail page banner. |
| Gallery photos | `public/media/communities/<slug>/` | Mixed | 4-6 photos typically. Mix of aerials, amenities, lifestyle shots. |
| Plat map PDF | `public/documents/plat-maps/` | N/A | The downloadable PDF. Named `<slug>-<division>.pdf`. |
| Plat map preview JPG | `public/media/plat-maps/` | Varies | A JPG render of the plat for visual preview. Named `<slug>-<division>-preview.jpg`. |

Reference paths in `communities.ts` are absolute from the public root.

## Plat maps

Each community can have multiple plat maps (one per division). They appear on the detail page as clickable buttons that open the PDF in a new tab.

To add a new plat map:

1. Save the PDF to `public/documents/plat-maps/<slug>-<label>.pdf`
2. Generate a preview JPG (just a render of page 1 at ~1000px wide) and save to `public/media/plat-maps/<slug>-<label>-preview.jpg`
3. Add an entry to the community's `platMaps` array
4. The `label` field is what shows on the button — keep it short ("Division 4", "Phase 2")
5. `fileSizeMB` is just for the download UI; eyeball it from `ls -la`

## Common operations

### Add a new community

1. Confirm with Curtis: name, slug, status, location, what makes it distinct, photos available
2. Get the assets — card image, hero, gallery photos, plat maps
3. Save assets to the locations above
4. Add the community object to `communities.ts`
5. If the community has available lots, also add lot rows to `lots.ts` (or seed them via Apps Script — see `apps-script-ops` skill)
6. Build + deploy

### Edit an existing community

Find it in the array, edit the field. Common edits:
- `description` / `longDescription` — marketing copy refinements
- `image` / `heroImage` — new photos from Curtis
- `platMaps` — when a new division gets recorded, add a new platMap entry
- `status` — flip from `'coming-soon'` to `'selling'` when lots release

### Remove a community

Same caveat as plans — check for references first:

```bash
grep -r "<community-slug>" /Users/buildguardian/Code/odyssey-site/src/
```

If lots reference the community via `communitySlug`, decide:
- If the community is built out and sold → remove the community entry, leave the lot rows (they'll show as Sold and get auto-filtered)
- If the community is genuinely discontinued → remove the community AND its lots

## Naming + slugging conventions

- Slugs are lowercase, hyphenated, match the community brand name exactly: `granite-creek`, `hawks-landing`, `sand-creek-estates`.
- Once a community is live, **don't change its slug** — breaks bookmarks, breaks Realtor links, breaks plat map references.
- For renaming: add a redirect in `public/_redirects` from old slug to new.

## How communities pair with the lot data

Each row in `src/data/lots.ts` (or the master Lot Inventory Google Sheet, once that's wired in) has a `communitySlug` that MUST match a community's `slug`. If a lot references `granite-creek` but no community has that slug, the lot won't render anywhere (no filtering bucket exists).

Conversely, communities can exist with zero lots — they just won't show up on the `/available-lots/` index until lots are added.
